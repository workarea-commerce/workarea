# WA-009 — Fix Redis `sadd`/`sadd?` deprecation warnings

## Summary

`redis` gem v4 defaults to `Redis.sadd_returns_boolean = true`, which causes a deprecation warning when `Redis#sadd` is called with a single member:

> `Redis#sadd will always return an Integer in Redis 5.0.0. Use Redis#sadd? instead.`

Some dependencies (e.g. Sidekiq extensions) call `sadd` with a single member, which triggers the warning during app/test boot.

To eliminate the warning (and align behavior with Redis 5), Workarea now opts into the Redis 5 return type globally during engine boot:

```ruby
Redis.sadd_returns_boolean = false
```

## Change made

- `core/lib/workarea/core/engine.rb`
  - In `config.before_configuration`, require `redis` and set:
    - `Redis.sadd_returns_boolean = false`

## Notes

- This avoids patching any gems under `vendor/bundle`.
- This changes the return value of `Redis#sadd` for single-member adds from boolean → integer, matching Redis 5 behavior.
