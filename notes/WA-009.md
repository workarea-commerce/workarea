# WA-009 — Deprecation burn-down: Redis `sadd?` warnings

## Goal
Eliminate Redis `sadd?` deprecation warnings (Redis 4.8+ warning that Redis 5 will always return an Integer).

## Root cause
Redis 4.x includes a compatibility mode where `sadd` can return booleans; it emits deprecation warnings indicating Redis 5 will always return Integers.

Workarea does not rely on boolean return values from `SADD`.

## Fix
Opt into Redis 5 behavior at boot, prior to creating the persistent Redis client:

- Set `Redis.sadd_returns_boolean = false` inside `Workarea.redis`.

File changed:
- `core/lib/workarea.rb`

## Notes
This change is intentionally minimal and centralized: Workarea’s Redis client factory is the safest place to set the global Redis compatibility toggle.
