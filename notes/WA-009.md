# WA-009 — Deprecation burn-down: Redis `sadd?` warnings

## Goal
Eliminate Redis `sadd?` deprecation warnings (Redis 4.8+ warning that Redis 5 will always return an Integer).

## Root cause
Redis 4.x includes a compatibility mode where `sadd` can return booleans; it emits deprecation warnings indicating Redis 5 will always return Integers.

Workarea does not rely on boolean return values from `SADD`.

## Fix
Opt into Redis 5 behavior at boot (as early as possible), before any Redis clients are created:

- In `core/lib/workarea/core/engine.rb` within `config.before_configuration`, `require 'redis'` and set:
  - `Redis.sadd_returns_boolean = false`

Also removed a redundant/too-late assignment from `Workarea.redis`.

Files changed:
- `core/lib/workarea/core/engine.rb`
- `core/lib/workarea.rb`

## Verification

## Notes
This change is intentionally minimal and centralized: the Rails engine’s `before_configuration` hook runs early enough to avoid any Redis client initialization happening before the global compatibility toggle is set.
