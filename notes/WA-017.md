# WA-017: ES 7.x Mapping Type Removal (Core)

**Date:** 2026-02-12  
**Status:** Complete  
**Branch:** modernization/upgrade-platform-2026-02  
**Related:** WA-011 (Performance Test Suite - Elasticsearch 7.x Compatibility)

## Summary

Removed all `_doc` mapping type wrappers and deprecated field types from Elasticsearch index definitions and queries in `core/`. ES 7.x removed mapping types entirely - any requests with explicit type parameters (other than `_doc`) return errors. This was the root cause of 365+ storefront test failures and all performance test failures.

## Changes Made

### 1. Configuration - ES Mapping Definitions
**File:** `core/lib/workarea/configuration.rb`

**Removed type wrappers from all mappings:**
- `admin` mapping: Removed nested `admin: { ... }` wrapper
- `storefront` mapping: Merged `category` and `storefront` type mappings into single definition
- `help` mapping: Removed nested `help: { ... }` wrapper
- `order` mapping: Removed nested `order: { ... }` wrapper

**Updated deprecated field types:**
- `type: 'string'` → `type: 'text'` or `type: 'keyword'` (context-dependent)
- Removed `index: 'not_analyzed'` (use `type: 'keyword'` instead)
- Removed `analyzer: 'keyword'` from keyword field dynamic templates

**Storefront mapping notes:**
- Added `query: { type: 'percolator' }` field to main properties (previously in separate `category` type)
- Added `release_id` and `changeset_release_ids` fields for category query support

### 2. Elasticsearch Document Operations
**File:** `core/lib/workarea/elasticsearch/document.rb`

Changed all ES operation type parameters from dynamic `type: type` to fixed `type: '_doc'`:
- `save` method
- `bulk` method
- `update` method
- `delete` method
- `count` method
- `search` method

### 3. Total Count Format Handling
**Files:**
- `core/lib/workarea/elasticsearch/index.rb`
- `core/app/queries/workarea/search/query.rb`

Added logic to handle ES 7.x's new total count format which returns `{ value: N, relation: "eq" }` instead of a plain integer:

```ruby
total = response['hits']['total']
# ES 7.x returns { value: N, relation: "eq" }, older versions return integer
total.is_a?(Hash) ? total['value'] : total
```

### 4. Category Query Operations
**File:** `core/app/models/workarea/search/storefront/category_query.rb`

Changed type parameters in direct index operations:
- `delete`: `type: 'category'` → `type: '_doc'`
- `save` (create_live): `type: 'category'` → `type: '_doc'`
- `save` (create_releases): `type: 'category'` → `type: '_doc'`

### 5. More Like This (MLT) Query Syntax
**Files:**
- `core/app/queries/workarea/search/related_products.rb`
- `core/app/queries/workarea/search/related_help.rb`

Updated MLT query syntax for ES 7.x compatibility. The `ids` parameter was removed in ES 7.x and replaced with `like` parameter using document references:

**Before (ES 5.x/6.x):**
```ruby
more_like_this: {
  ids: search_product_ids,
  like_text: like_text
}
```

**After (ES 7.x):**
```ruby
like_docs = search_product_ids.map { |id| { _id: id } }
like_docs << like_text if like_text.present?

more_like_this: {
  like: like_docs
}
```

### 6. Query Parameter Update
**File:** `core/app/queries/workarea/search/product_search.rb`

Changed deprecated bool query parameter:
- `minimum_number_should_match: 1` → `minimum_should_match: 1`

### 7. Test File Updates
**File:** `core/test/lib/workarea/elasticsearch/document_test.rb`

- Removed type wrapper from test mapping definition
- Updated all explicit `type: 'foo'` and `type: 'bar'` references to `type: '_doc'`

## Migration Notes

### For Plugin/Extension Developers

If you have custom Elasticsearch mappings, update them as follows:

**Old format:**
```ruby
config.elasticsearch_mappings.my_type = {
  my_type: {
    properties: { ... }
  }
}
```

**New format:**
```ruby
config.elasticsearch_mappings.my_type = {
  properties: { ... }
}
```

### Field Type Updates

| Old | New |
|-----|-----|
| `type: 'string', index: 'not_analyzed'` | `type: 'keyword'` |
| `type: 'string', analyzer: '...'` | `type: 'text', analyzer: '...'` |
| `type: 'keyword', analyzer: 'keyword'` | `type: 'keyword'` (remove analyzer) |

### Direct Index Operations

If your code calls index operations directly (not through the Document interface), ensure you pass `type: '_doc'`:

```ruby
# Wrong (ES 7.x will error)
index.save(document, type: 'my_type')

# Correct
index.save(document, type: '_doc')
```

### Total Count Extraction

If your code extracts total counts from ES responses, handle both formats:

```ruby
total = response['hits']['total']
total = total.is_a?(Hash) ? total['value'] : total
```
