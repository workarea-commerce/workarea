## Analysis

### Nature of Errors

All 365 errors appear to be **Elasticsearch 7.x compatibility issues** (documented in WA-011). The fact that there are:
- **0 failures** (test assertions passed)
- **365 errors** (environment/setup issues)

...strongly suggests these are infrastructure-related, not logic failures.

### Representative Error Messages

All 365 errors share the same root cause: **Elasticsearch 7.x removed mapping types**, but Workarea's mapping definitions still use the old nested type structure.

**Error Pattern:**
```

  "error": {

**Stack Trace:**
```

**Root Cause:** Mapping definitions in `core/lib/workarea/configuration.rb` wrap properties in type names (`admin`, `storefront`, etc.), but ES 7.x only accepts direct property definitions under `_doc`.

**Impact:** Every test triggering index creation fails in the `before_setup` hook, preventing test execution.

**Example Affected Tests:**
- `Workarea::Storefront::SitemapsIntegrationTest#test_viewing_robots_txt`
- `Workarea::Storefront::HealthCheckIntegrationTest#test_health_check`
- (363+ additional tests with identical error)

This aligns with the known ES 7.x compatibility issues where:
- Index creation/mapping fails

- Tests abort during setup phase

### ChromeDriver Resolution

**Problem:** System tests initially failed to start due to ChromeDriver version incompatibility with Chrome 115+.

**Root Cause:** The `webdrivers` gem (v4.x) uses deprecated Google download endpoints that don't support Chrome 115+ (Chrome for Testing migration). Auto-download failed with version resolution errors.

**Solution:** (Git commit `788ca511`, 2026-02-09)
- Bypassed the `webdrivers` gem's auto-download mechanism
- Pointed Selenium at locally-installed ChromeDriver via `CHROMEDRIVER_PATH` or system PATH
- Ensured version match: Chrome 145.0.7632.46 ↔ ChromeDriver 145.0.7632.46

**Code Change:** Modified `testing/lib/workarea/system_test.rb`:
```ruby
chromedriver_path = ENV.fetch('CHROMEDRIVER_PATH') {
  `which chromedriver 2>/dev/null`.strip.presence
}
if chromedriver_path
  Webdrivers::Chromedriver.define_singleton_method(:update) { chromedriver_path }
  Selenium::WebDriver::Chrome::Service.driver_path = chromedriver_path
end
```

**Verification:**
- ✅ Chrome and ChromeDriver versions match (145.0.7632.46)
- ✅ System tests launch headless Chrome successfully
- ✅ All system test failures are ES-related (not driver issues)

**Long-term:** Upgrade to `webdrivers` v5.x (supports Chrome for Testing) or migrate to Selenium Manager (Selenium 4.6+).
