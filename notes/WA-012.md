## Issues encountered & fixes

### 1) Rails boot failure: `uninitialized constant ActiveSupport::LoggerThreadSafeLevel::Logger`
**Error:**
```
NameError: uninitialized constant ActiveSupport::LoggerThreadSafeLevel::Logger
... activesupport-6.1.7.10/lib/active_support/logger_thread_safe_level.rb
```

**Cause:** In this dummy app we load a minimal set of railties, and `Logger` (Ruby stdlib) wasn’t required early enough. ActiveSupport assumes the `Logger` constant exists.

**Fix:** Require Ruby’s stdlib logger during boot.
- Updated: `admin/test/dummy/config/boot.rb`
- Added:
  ```rb
  require 'logger'
  ```

### 2) Teaspoon/Selenium failure: webdrivers can’t resolve chromedriver for modern Chrome
**Error:**
```
Webdrivers::VersionError: Unable to find latest point release version for 145.0.7632...
404 https://chromedriver.storage.googleapis.com/LATEST_RELEASE_145.0.7632
```

**Cause:** Chrome 115+ uses **Chrome for Testing (CfT)** endpoints; the `webdrivers` gem (4.x) still queries the legacy `chromedriver.storage.googleapis.com` endpoints and fails for modern/beta Chrome versions.

**Fix:** Disable webdrivers auto-update and point Selenium to the locally-installed chromedriver (same approach as `testing/lib/workarea/system_test.rb`).
- Updated: `testing/lib/workarea/testing/teaspoon.rb`
- Added logic to use `ENV['CHROMEDRIVER_PATH']` or `which chromedriver`, then:
  - override `Webdrivers::Chromedriver.update`
  - set `Selenium::WebDriver::Chrome::Service.driver_path`
