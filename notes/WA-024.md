# WA-024: BSON deep_symbolize_keys Deprecation Cleanup

**Date:** 2026-02-13  
**Status:** ✅ Complete  
**Ruby Version:** 2.7.8  
**BSON Version:** 5.2.0

## Problem

BSON 5.x deprecates calling `deep_symbolize_keys!` directly on `BSON::Document` instances and will raise an error in v6. The deprecation message states:

```
[DEPRECATION] `deep_symbolize_keys!` is not supported on BSON::Document instances.
Please convert the document to a Hash first (using `#to_h`), then call `#deep_symbolize_keys!` on the Hash.
This will raise an error starting with the v6.0.0 release.
```

### Root Cause

The warnings were coming from the `money-rails` gem (v1.13.4) in its Mongoid integration:

```ruby
# vendor/bundle/ruby/2.7.0/gems/money-rails-1.13.4/lib/money-rails/mongoid/money.rb:52
def mongoize_hash(hash)
  hash.deep_symbolize_keys! if hash.respond_to?(:deep_symbolize_keys!)
  # ...
end
```

When Mongoid reads Money fields from MongoDB, it passes `BSON::Document` objects to this method, which then calls the deprecated method.

## Solution

Instead of patching the `money-rails` gem (which would be fragile and difficult to maintain), we patched `BSON::Document` itself to handle the deprecation transparently.

### Changes Made

**File:** `core/lib/workarea/ext/freedom_patches/bson_document.rb` (NEW)

```ruby
if defined?(BSON::Document)
  class BSON::Document
    if method_defined?(:deep_symbolize_keys!)
      alias_method :original_bson_deep_symbolize_keys!, :deep_symbolize_keys!
      
      def deep_symbolize_keys!
        # Convert to regular Hash to avoid BSON deprecation
        hash = to_h
        # Call deep_symbolize_keys! on the Hash
        hash.deep_symbolize_keys!
        # Replace our contents with the symbolized hash
        replace(hash)
        self
      end
    end
  end
end
```

**File:** `core/lib/workarea/core.rb` (MODIFIED)

Added require for the new freedom patch:

```ruby
require 'workarea/ext/freedom_patches/bson_document'
```

## Why This Approach?

1. **Transparent:** Works for ALL code that calls `deep_symbolize_keys!` on BSON::Document, not just money-rails
2. **Maintainable:** No need to track changes in third-party gems
3. **Forward-compatible:** Prepares codebase for BSON 6.x where this will raise errors
4. **Minimal impact:** Only affects the deprecated code path

## Testing

bundle exec rake core_test 2>&1 | rg "\[DEPRECATION\] \`deep_symbolize_keys" | wc -l

### After Fix
```bash
bundle exec rake core_test 2>&1 | rg "\[DEPRECATION\] \`deep_symbolize_keys" | wc -l
# Result: 0 deprecation warnings
```

## Alternatives Considered

1. **Patch money-rails directly:** Would require overriding private methods and be fragile
2. **Fork money-rails:** Too much maintenance burden
3. **Replace money-rails:** Too large a scope for this issue

## Future Work

Monitor for BSON 6.x release and verify this patch continues to work or if the method is removed entirely (in which case our override would safely do nothing).
