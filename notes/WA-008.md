## Objective

Eliminate Sidekiq deprecation warnings by removing the `default_worker_options` compatibility shim and modernizing to use `default_job_options` (Sidekiq 7+ convention).

## Changes Made

### File Modified
- `core/lib/workarea/configuration/sidekiq.rb`

### What Changed
Removed the compatibility shim (lines 73-82) that provided backward compatibility for `default_worker_options`:

```ruby
# REMOVED:
# Sidekiq 6.5 deprecates `default_worker_options` in favor of
# `default_job_options`, but still calls it internally.
# Provide a shim to avoid noise and prepare for Sidekiq 7.
if ::Sidekiq.respond_to?(:default_job_options) &&
    ::Sidekiq.respond_to?(:default_job_options=)
  ::Sidekiq.singleton_class.class_eval do
    define_method(:default_worker_options) { default_job_options }
    define_method(:default_worker_options=) { |opts| self.default_job_options = opts }
  end
end
```

### Why This Works
- Sidekiq 6.5+ already provides `default_job_options` natively
- The shim was only needed for transitional compatibility
- No Workarea code actually uses these methods directly
- Vendor gems (sidekiq-unique-jobs) that use deprecated methods will trigger their own deprecations, not ours
