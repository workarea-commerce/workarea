# WA-013: Storefront JavaScript tests (Teaspoon)

## Goal
Run `bundle exec rake storefront_teaspoon` and fix failures.

## Initial failure
Running `bundle exec rake storefront_teaspoon` failed early with:

- `uninitialized constant ActiveSupport::LoggerThreadSafeLevel::Logger (NameError)`

This is the Rails 6.1 + Ruby 2.6+/stdlib `Logger` load-order issue.

## Fixes
1. **Load `logger` early in the storefront dummy app boot**
   - File: `storefront/test/dummy/config/boot.rb`
   - Added `require 'logger'` before ActiveSupport/rails boots.

2. **Make blur/focusout validation explicit in JS specs**
   - File: `storefront/test/javascripts/forms_spec.js`
   - The validation is configured via jQuery validate `onfocusout`, but the specs were only triggering `focus` on the next element. In the test environment, this does not reliably trigger a focusout/blur event on the prior element.
   - Updated specs to explicitly `trigger('focusout')` when simulating leaving a field.

3. **Chromedriver resolution for Chrome 115+ (shared Teaspoon harness)**
   - File: `testing/lib/workarea/testing/teaspoon.rb`
   - Added logic to bypass `webdrivers` auto-update and point Selenium at a locally installed `chromedriver` (or `CHROMEDRIVER_PATH`).
   - This matches the WA-012 fix and ensures storefront Teaspoon runs on modern Chrome.

## Result
After fixes:

- `bundle exec rake storefront_teaspoon`
- **61 examples, 0 failures**
