# WA-015: ActiveModel::Errors Deprecation Fix

**Issue:** `ActiveModel::Errors#<<` method deprecated in Rails 6.1, removed in Rails 7
**Location:** `core/lib/workarea/validators/parameter_validator.rb:4`
**Status:** ✅ Fixed

## Change

Replaced deprecated `errors[attribute] << "message"` with `errors.add(attribute, "message")`.

### Before
```ruby
record.errors[attribute] << <<-eos
  must contain only alphanumeric, underscore, and hyphen characters
eos
```

### After
```ruby
record.errors.add(attribute, "must contain only alphanumeric, underscore, and hyphen characters")
```

## Rails 7 Compatibility

This change ensures compatibility with Rails 7, where the `<<` method has been completely removed from `ActiveModel::Errors`. The `add` method is the standard approach for adding validation errors in Rails 6.1+ and Rails 7.

## Notes

- The heredoc syntax was also simplified to a simple string since it was only one line of text
- No functional changes to validation logic
- Error message remains identical
